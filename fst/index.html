FST 记录

发表于 2018-07-17 | 分类于 Algorithms |

有一天，突然发现自己的代码能力可能不如一年之前。
于是决定做一些训练，在这里记录写题的失误（以 FST 为主）。

  • 模数为质数的情况下，欧拉定理会在底数为 0 时失效。
  • 某环上的元素按顺时针编号为 [0, n)。用 [l, r) 表示从 l 顺时针走到 r 经过的区间（含 l，不含 r）。将区间按照右端点排序的时候，没有考虑右端点为 0 的区间。
  • 某序列长度为 2N。定义变量 n = 2N。试图把 2n 作为序列长度。
    • UPD：测极限数据的时候，用 -fsanitize=address -ftrapv -O2 测一遍，再用 -O2 测一遍。
  • 在字符串上操作高精度数时，注意字符串输入后要左右翻转。
  • 树链剖分 + 递归线段树。线段树使用右开区间。一条重链的右边界应该是重链底的编号 + 1。写的时候可能会忘记 + 1。
    • UPD：直接在重链顶记录下这条重链的右边界。
  • 使用 pair<int, int> 代替结构体时，注意不要混淆两个变量。
    • UPD：尽量自己写结构体。
  • 使用 -1 表示空指针的时候，注意避免对空指针的访问。
  • 德州扑克中计算得分部分写错了，原因未知，重构代码后解决。
    • UPD1：德州扑克中可以写一个函数，把五张牌按照重数为第一关键字、大小为第二关键字排序，哈希成一个整数。除了两种顺子之外的所有牌型都可以用这个哈希函数作为第二关键字。
    • UPD2：把 Ace 存成最大的牌，特判从 Ace 开始的顺子。
  • 写错模数。
